Class: Main

h5c3. Main

new Main()

[Extends h5c3.Base]

h5c3.Main is the primary base class for creating a game and drives resources, core processing (cycling) your game, and serves as a placeholder for scenes.

Basic Usage

Typically a h5c3.Main is constructed by the h5c3.start method call made from within your games index.html: 


<script>
   // h5c3.start will construct an instance of TheGame once the device (browser) is ready
   h5c3.start('waCANVAS', 'TheGame', '/mygame/js/', ['mygame.js']);
</script>
When the h5c3.start system has finished preparing everything, it will dynamically construct an instance of the class parameter (in the above example 'TheGame'). You can always gain access to the game from the global h5c3.device: 

var myGame = this;

To create a h5c3.Main, extend it and override what you need: 


TheGame = h5c3.Main.extend('TheGame',
{},
{
    // onReady is called when the browser DOM is ready
    onReady:function ()
    {
        this._super();

        // load resources
        // declare a base URL; saves you typing
        h5c3.device.loader.setBaseUrl('images/');

        // add an image to the resource loader's queue
        h5c3.device.loader.add(new h5c3.Image('player-ship', 'ship1.png'));

        // start the resource loader
        h5c3.device.loader.start(this.onLoading.bind(this), this.onLoaded.bind(this));
    },

    onLoading:function (percentageComplete)
    {
        // draw title screen -- with loading bar
    }

});
See the h5c3.Loader for more information on using the resource loader and the onLoading/onLoaded callbacks.

Pause/Resume

You can pause/resume individual scenes, or you can pause/resume all scenes by calling pause on the game: 


myGame.pause();
myGame.resume();
myGame.togglePauseResume();
Debugging
h5c3.Main sets up the following default input keys for debugging:
  • F8 to enable/disable showing development window.
  • F9 to enable/disable physics debugging across all layers.
  • F10 to dump stats on the object pools.
  • F11 toggle sound.

h5c3. Main

new Main()

[Extends h5c3.Base]

h5c3.Main is the primary base class for creating a game and drives resources, core processing (cycling) your game, and serves as a placeholder for scenes.

Basic Usage

Typically a h5c3.Main is constructed by the h5c3.start method call made from within your games index.html: 


<script>
   // h5c3.start will construct an instance of TheGame once the device (browser) is ready
   h5c3.start('waCANVAS', 'TheGame', '/mygame/js/', ['mygame.js']);
</script>
When the h5c3.start system has finished preparing everything, it will dynamically construct an instance of the class parameter (in the above example 'TheGame'). You can always gain access to the game from the global h5c3.device: 

var myGame = this;

To create a h5c3.Main, extend it and override what you need: 


TheGame = h5c3.Main.extend('TheGame',
{},
{
    // onReady is called when the browser DOM is ready
    onReady:function ()
    {
        this._super();

        // load resources
        // declare a base URL; saves you typing
        h5c3.device.loader.setBaseUrl('images/');

        // add an image to the resource loader's queue
        h5c3.device.loader.add(new h5c3.Image('player-ship', 'ship1.png'));

        // start the resource loader
        h5c3.device.loader.start(this.onLoading.bind(this), this.onLoaded.bind(this));
    },

    onLoading:function (percentageComplete)
    {
        // draw title screen -- with loading bar
    }

});
See the h5c3.Loader for more information on using the resource loader and the onLoading/onLoaded callbacks.

Pause/Resume

You can pause/resume individual scenes, or you can pause/resume all scenes by calling pause on the game: 


myGame.pause();
myGame.resume();
myGame.togglePauseResume();
Debugging
h5c3.Main sets up the following default input keys for debugging:
  • F8 to enable/disable showing development window.
  • F9 to enable/disable physics debugging across all layers.
  • F10 to dump stats on the object pools.
  • F11 toggle sound.

h5c3. Main

new Main()

[Extends h5c3.Base]

h5c3.Main is the primary base class for creating a game and drives resources, core processing (cycling) your game, and serves as a placeholder for scenes.

Basic Usage

Typically a h5c3.Main is constructed by the h5c3.start method call made from within your games index.html: 


<script>
   // h5c3.start will construct an instance of TheGame once the device (browser) is ready
   h5c3.start('waCANVAS', 'TheGame', '/mygame/js/', ['mygame.js']);
</script>
When the h5c3.start system has finished preparing everything, it will dynamically construct an instance of the class parameter (in the above example 'TheGame'). You can always gain access to the game from the global h5c3.device: 

var myGame = this;

To create a h5c3.Main, extend it and override what you need: 


TheGame = h5c3.Main.extend('TheGame',
{},
{
    // onReady is called when the browser DOM is ready
    onReady:function ()
    {
        this._super();

        // load resources
        // declare a base URL; saves you typing
        h5c3.device.loader.setBaseUrl('images/');

        // add an image to the resource loader's queue
        h5c3.device.loader.add(new h5c3.Image('player-ship', 'ship1.png'));

        // start the resource loader
        h5c3.device.loader.start(this.onLoading.bind(this), this.onLoaded.bind(this));
    },

    onLoading:function (percentageComplete)
    {
        // draw title screen -- with loading bar
    }

});
See the h5c3.Loader for more information on using the resource loader and the onLoading/onLoaded callbacks.

Pause/Resume

You can pause/resume individual scenes, or you can pause/resume all scenes by calling pause on the game: 


myGame.pause();
myGame.resume();
myGame.togglePauseResume();
Debugging
h5c3.Main sets up the following default input keys for debugging:
  • F8 to enable/disable showing development window.
  • F9 to enable/disable physics debugging across all layers.
  • F10 to dump stats on the object pools.
  • F11 toggle sound.
